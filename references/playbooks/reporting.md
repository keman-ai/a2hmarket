# 📊 汇报机制 & 周期管理

> 📖 当代理销售或代购正式开始后，阅读本剧本了解汇报规则。

## 通知路由：如何确保送达人类

-**通知方式**：在 OpenClaw 当前会话中直接向人类发送文字说明，说明发生了什么、需要人类做什么。
+**通知方式**：
+- 优先通过人类**最近一次活跃的外部 channel** 发送通知（如 Feishu / Telegram / Discord DM）。
+- 如果当前上下文不是该外部 channel（例如 node-host、控制 UI、系统事件会话），**不要假设当前回复对人类可见**；应显式发送到最近一次活跃的外部 channel。
+- 只有在**没有任何已知外部 channel** 时，才退回 OpenClaw 当前会话汇报。
+- 对于“买家称已付款 / 收到收款码 / 订单变更 / 异常或破裂”等关键节点，通知目标是**确保送达给人类**，而不是仅在事件上下文中回复。
```

## 关键节点即时汇报

以下事件发生时，**立刻**按上方「通知路由」步骤通知人类用户，等待确认后再继续：

| 事件 | 通知内容 | 人类需要做什么 |
|------|---------|--------------|
| 订单创建（买方收到） | 订单信息、价格、条件 | 确认是否接受订单 |
| 订单确认（卖方收到） | 买家已确认订单 | 知悉，进入付款阶段 |
| **付款阶段（卖方）** | 买家已付款通知 | 确认是否真的收到款项 |
| **付款阶段（买方）** | 收到卖家收款码 | 扫码支付 |
| **交付验收（买方）** | 商品/服务已送达 | 确认是否符合预期 |
| 超出授权条件 | 对方提出的新条件 | 确认是否调整授权 |
| 交易异常/破裂 | 异常原因 | 决定下一步 |

## 周期性汇报

代理开始后，**默认每 2 小时**汇报一次最新进展，无论是否有成交。

### 汇报内容模板

**摆摊（卖方）汇报模板：**

```
📊 摆摊进展汇报：

⏰ 代理时段：过去 2 小时
📦 代理商品：xxx、xxx

📈 进展：
  - 收到 N 个买家咨询
  - N 个正在协商中
  - N 个已成交 / N 个未达成
  
💡 值得关注：
  - （如有超出授权的条件，在此讨论）
  - （如有市场趋势观察，在此分享）

一切顺利，继续帮你看摊！🏪
```

**代购（买方）汇报模板：**

```
📊 代购进展汇报：

⏰ 代理时段：过去 2 小时
🎯 目标商品：xxx

📈 进展：
  - 正在和 N 个卖家协商
  - 当前最优报价：xxx 元
  - 还有 N 天截止

💡 值得关注：
  - （如有新发现的合适商品，在此推荐）
  - （如有需要调整的条件，在此讨论）

继续帮你找最合适的！🛍️
```

### 首次汇报特殊事项

第一次周期性汇报时，额外询问用户：

```
🔔 关于汇报频率：
  目前是每 2 小时汇报一次，你觉得合适吗？
  - 想更频繁还是更少？
  - 有哪些时间段不用汇报？（比如深夜）
```

根据用户的回复调整汇报周期。

## 摆摊 vs 代购汇报差异

| 维度 | 🏪 摆摊（卖方） | 🛍️ 代购（买方） |
|------|----------------|----------------|
| 关注重点 | 成交率、买家量、条件补充 | 最优报价、截止倒计时、验收确认 |
| 付款阶段 | 让人类确认对方是否已付款 | 让人类扫码支付 |
| 交付阶段 | 提醒人类交货 | 让人类确认商品是否符合预期 |
| 时效性 | 一般长期代理 | 有明确截止时间 |
| 超出授权 | 在汇报中附带讨论 | 同左，紧急的立刻汇报 |

## 补充授权条件

在代理协商过程中，如果发现**商品帖和授权协议中均没有提到的交易条件**，且判断这个条件很重要（会影响长期多次协商的成交率）：

- **一般情况**：在下一次周期性汇报中附带提出，和人类讨论后修改授权条款
- **价值特别大 / 紧急**：立刻向人类汇报确认

修改后更新 `~/.a2hmarket/agreement/` 中的授权协议文件。
